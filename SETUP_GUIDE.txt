# PrintRUSH Lopez — Setup & Deployment Guide

I have already updated your `js/config.js` with the Supabase credentials you provided. To fully publish and deploy the project, follow these steps:

## 1. Database Setup (Supabase)

You must initialize your database schema and create your first shop.

### Step A: Initialize Schema
1. Log in to your Supabase Dashboard (https://supabase.com/dashboard).
2. Go to your project (iovsadqmwnjssrcxvagu).
3. Open the SQL Editor in the left sidebar.
4. Create a "New Query".
5. Copy the entire contents of `supabase/schema.sql` and paste it into the editor.
6. Click Run. This will create all tables, indexes, and functions (including the PostGIS map search).

### Step B: Create Your First Admin & Shop
Before creating shops, you need to promote your account to a Platform Admin.
1. Go to your project's `/owner/login.html` and sign up with the email you want to use.
2. Go to the **Authentication** section in your Supabase Dashboard and copy your **User ID** (UUID).
3. Open the SQL Editor in Supabase and run this command:

```sql
-- Replace 'PASTE_YOUR_USER_ID_HERE' with your UUID from Supabase Auth
INSERT INTO platform_admins (user_id, role) 
VALUES ('PASTE_YOUR_USER_ID_HERE', 'super_admin');
```

4. Now, visit your hidden Admin Dashboard using your secret easter egg URL:
`https://print-rush-lopez.vercel.app/admin/?code=printrush-admin-easteregg` (Replace with your actual live URL)
5. From the Admin Dashboard, use the **Shop Management** form to easily create your first shop and get your **Shop ID**.

## 3. Edge Functions & Sandbox Integrations

This project now uses a fully internal sandbox for payments and delivery simulation. That means there is no need for a real Shipmates or PayMongo account to launch a demo.

### Prerequisites
1. You must have the Supabase CLI installed on your computer.
2. Open your terminal in the `PrintRush Lopez` project folder.
3. Log in to the CLI: `supabase login`
4. Link your project: `supabase link --project-ref iovsadqmwnjssrcxvagu`

### Step A: Deploy Sandbox Edge Functions
Deploy the sandbox functions that handle payments, delivery booking, and status updates:
```bash
supabase functions deploy create-payment
supabase functions deploy dispatch-delivery
supabase functions deploy mock-payment-status
supabase functions deploy delivery-status
```

### Step B: Sandbox Payment and Delivery
- No live `SHIPMATES_API_KEY` is required.
- No live `PAYMONGO_SECRET_KEY` is required.
- The platform now stores sandbox `payments` and `deliveries` records in Supabase with simulated timelines.
- The checkout flow is handled with a local `/payment.html` sandbox experience.

### Step C: Optional Real Push Notifications
Web Push is still optional. If you want notifications in the browser, deploy the `notify-push` function and configure VAPID keys.
1. Deploy the function:
```bash
supabase functions deploy notify-push
```
2. Save the VAPID keys via CLI:
```bash
supabase secrets set VAPID_PRIVATE_KEY=your_vapid_private_key_here
supabase secrets set VAPID_PUBLIC_KEY=your_vapid_public_key_here
```
3. Update `js/lib/push.js` with the public VAPID key.

### Step D: Configure hCaptcha (Anti-Spam)
hCaptcha prevents bots from uploading massive files and filling up your storage.
1. Create a free account at https://www.hcaptcha.com/.
2. Add a new site (e.g., print-rush-lopez.vercel.app) to get a Sitekey.
3. Open `order.html` and locate the `<div class="h-captcha" ...>` on line ~331.
4. Replace the `data-sitekey="10000000-ffff-ffff-ffff-000000000001"` with your actual Sitekey.
5. Get your Secret Key from your hCaptcha account settings.
6. Save the Secret Key to your Supabase project securely via CLI (used by the backend to verify the captcha):
```bash
supabase secrets set HCAPTCHA_SECRET=your_hcaptcha_secret_key_here
```
